# 正式发布验收清单

本文档用于对一个确定的候选版本逐项签字。它不替代自动化测试；它专门覆盖需要真实 Windows、Codex Desktop、微信账号、网络和发布凭证的边界。

- 发布到 `next` 的预览版可以保留明确标注的未完成实机项，但必须在 Release Notes 中列出。
- 发布到 `latest` 的稳定版不得保留“待验收”“未知”或未签字的必选项。
- 带“外部实机/凭证”的项目不能用单元测试、Mock、代码审查或 CI 结果代签。
- 证据只记录版本、时间、脱敏日志、截图编号或任务 ID；不得粘贴 Token、二维码、媒体密钥、完整本机路径或消息正文。

相关依据：[SPEC](../SPEC.md)、[发布与分发流程](./npm-publishing.md)、[安全政策](../SECURITY.md)。

## 外部前置条件

以下资源由发布负责人提供，仓库和 CI 无法自行取得：

- [ ] 可还原的 Windows 10 x64 与 Windows 11 x64 干净环境；
- [ ] 已登录的 Codex Desktop，至少覆盖最低版本 `0.144.2` 和一个已验证的 `0.144.x` 版本；
- [ ] 一个真实微信控制者账号，以及用于验证越权隔离的第二账号和群聊；
- [ ] 可模拟超时、断网、Bridge 重启、锁屏和离开状态的实机环境；
- [ ] npm 包名所有权已完成一次性 bootstrap 引导；bootstrap 版本只在 `bootstrap` tag、已 deprecated，临时发布凭证已撤销；
- [ ] npm `codex-ilink` 已把 `Obito-404/Codex_iLink` 的 `release.yml` 配置为 trusted publisher，且 Environment 约束覆盖目标通道；
- [ ] GitHub 已配置 `preview-release` / `stable-release` Environment，稳定环境带 required reviewers；
- [ ] `stable-release` 已配置 Authenticode PFX 两项 Secret；预览版不要求该凭证。
